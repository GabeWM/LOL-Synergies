Introduction¶

League of Legends (or commonly refered to as League for short) was created ten years ago by Riot Games and has over 50 million daily active users. This makes it the most popular PC game of all time and it is played casually and competively all over the world. Given its popularity there is also a huge esports scene and many are even pushing for esports and league specfically to be added to the olympics. I personally have played league on and off for over 5 years and although there is a very high learning curve, it can be toxic, and is quite addicting I have enjoyed my time playing this game like many others. Given that is a team based game where 10 players in a 5v5 format select between 138 unique champions, (characters) certain synergies have risen based on their abilties working well together. I want to take a look at some of the best synergies from our dataset and the most popular synergies in the League community. I will investigate the winrate of these synergies and if these synergies work best in early game or late game team compositons.

Motivation¶

There are a few reasons why this information would be valuable, but I will describe a couple. If you are a player trying rank up it would be improtant to know when to pick which synergies depending on if your composition plays for early or late game. If you are a esports player/coach it is important to understand what kind of composition the you or the enemy is picking based on the synergies.

Background¶

League of Legends is very complicated game. I will try to define the relevant terms to this project below, but if you want to read more about it here is a link.

  • Seasons: Seasons refers to an interval of League of Legends defined by Riot Games. Major changes like new champions, reworks, and new items occur between seasons and they change annualy.
  • Patches: In each season there are many patches that cause minor changes to the game like buffs and nerfs.
  • Meta: Based on the current season and patch certain champions will be stronger than others. What is currently strong is refered to as the current meta.
  • Champions: A champions is just a selectable character. At the time of the dataset there are 138 champions for each player to choose from at the start of the game.
  • Champion Kits: A champion has a kit which just corresponds to their abilities.
  • Rank: League is a competitive game and has a ranking system. This varies from bronze all the way up to challenger. Players play games to climb up the ranks.
  • Crowd Control (CC): Crowd control refers to immobilizing effects like slows, stuns, and knockups.
  • Chain CC: Chain CC is when these crowd control effects are chained together.
  • Solo/Duo Lanes: There are 5 roles on each team. They are top lane, mid lane, support, attack damage carry and junguler. To make things simpiler I do not talk about positions except for the general categories solo lanes and duo lanes. A solo lane is like top and mid lane as you are the only person on your team in that lane. A duo lane is like bot lane where both the attack damage carry and support rescind. To make things simple just understand that you are in your lane during the early game.
  • Strong/Weak Champions and Synergies: Often champions or synergies will be refered to as strong or weak. This is based on the meta and people define if a champion/synergy is strong or weak by their winrate in that patch. I will also be using this logic.
  • Early game and Late game: League is a dynamic game and depending on the time of the game you are meant to be doing different things. I will refrain from discussing what these are as it is not needed, but I will make a simple distinction between early game (the beginning of the game) and late game (the end of the game). In the early game players are in their assigned lanes which means it is 1v1s and 2v2s depending on if it is a solo lane or duo lane. In the late game players teamfight which is a 5v5 battle.
  • Team Composition: Each team has 5 players. There composition is the 5 champions each player selected. Compositions can be drafted for a variety of strategies including playing for early game, late game, or even teamfights.

# In the end each champion will point to its winrate. 
# To get the winrates we will go through the dataframe and keep track of every game a champion is in and 
# every game a champion wins. We will then simply divide these values 
winrates = {}

# Iterate through the entire dataframe row by row
for index, row in tidy_df.iterrows():
    # Avoids code duplication as we are doing the same thing for each player and team
    for team in range(1,3):
        for i in range (1,6):   
            currChamp = tidy_df.at[index, 't' + str(team) + '_champ' + str(i)]
            
            # if the champion in question is already in the dictionary
            if (currChamp in winrates):
                totalWins = winrates[currChamp][0]
                totalGames = winrates[currChamp][1]
                
                # if the champion won the game
                if (tidy_df.at[index, 'winner'] == team):
                    winrates[currChamp] = (totalWins + 1, totalGames + 1)
                # if the champion lost the game
                else:
                    winrates[currChamp] = (totalWins, totalGames + 1)
            
            # if the champion in question is not in the dictionary yet
            else:
                # if the champion won the game
                if (tidy_df.at[index, 'winner'] == team):
                    winrates[currChamp] = (1, 1)
                # if the champion lost the game
                else:
                    winrates[currChamp] = (0, 1)
                    
# Now winrates holds a tuple of 2 elements where the first element is the number of wins and the second is the total
# number of games played for that champion. If we simply divide the two we will have our winrates
for key, value in winrates.items():
    winrates[key] = float(value[0])/float(value[1])

Now lets do the same thing, but for each combination of champions (synergy). Given that there are 138 champions at the time of this dataset and each champion can have a synergy with 137 other champions there are 18,906 different syrnegies. However this counts every synergy twice so in reality there will be 9,453 syrnegies. We are going to collect the winrates for all of them. However, if any synergy was played less than 175 times it will not be included in our analysis as there is not enough data on that synergy. This leaves us with 3,697 synergies. Remember each champion can only occur once in any game since these are ranked games.

# It will be easier if we build our datastructure first this time. In the end our datastucture will be a map 
# where each key is a tuple of the two champions that represent the synergy that then points to their winrate. 
winrates_synergy = {}

# Gets a sorted list of all the champions
list_of_champs = sorted(list(winrates.keys()))

# For each champion iterate through every champion
for i in list_of_champs:
    for j in list_of_champs:
        # If the champions are not the same and the synergy has not already been added to the dictionary
        if (i != j and ((i,j) not in winrates_synergy) and ((j,i) not in winrates_synergy)):
            # Again we will keep track of games won and total games played
            winrates_synergy[(i,j)] = (0, 0)

# Iterate through the entire dataframe row by row
for index, row in tidy_df.iterrows():
    # Avoids code duplication as we are doing the same thing for each player and team
    for team in range(1,3):
        for i in range (1,6):   
            currChamp1 = tidy_df.at[index, 't' + str(team) + '_champ' + str(i)]
            
            # For each possible synergy we must update our dictionary
            for j in range(1,6):
                currChamp2 = tidy_df.at[index, 't' + str(team) + '_champ' + str(j)]
                
                # Make sure we do not do anything if we are talking about the same champion
                if (i != j):
                    # To account for the fact that only one of (i, j) and (j , i) is in synergy
                    if (currChamp1 > currChamp2):
                        temp = currChamp1
                        currChamp1 = currChamp2
                        currChamp2 = temp
                    
                    totalWins = winrates_synergy[(currChamp1, currChamp2)][0]
                    totalGames = winrates_synergy[(currChamp1, currChamp2)][1]
                    
                    # if the champions won the game
                    if (tidy_df.at[index, 'winner'] == team):
                        winrates_synergy[(currChamp1, currChamp2)] = (totalWins + 1, totalGames + 1)
                    # if the champions lost the game
                    else:
                        winrates_synergy[(currChamp1, currChamp2)] = (totalWins, totalGames + 1)                     

# Now winrates holds a tuple of 2 elements where the first element is the number of wins and the second is the total
# number of games played for that synergy. If we simply divide the two we will have our winrates.
delete = []

for key, value in winrates_synergy.items():
    if (value[1] < 175):
        # Keeps track of the synergyies to delete if less than 175 games played
        delete.append(key)
    else:
        winrates_synergy[key] = float(value[0])/float(value[1])
        
# Deletes the synergyies with less than 175 games played
for i in delete:
    del winrates_synergy[i]

                # Make sure we do not do anything if we are talking about the same champion
                if (i != j):
                    # To account for the fact that only one of (i, j) and (j , i) is in synergy

avg = [{}, {}, {}, {}, {}]
